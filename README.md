# Loan Default / Credit Risk Scoring System

An explainable machine learning system that predicts whether a loan applicant is a good or bad credit risk — built to mirror how a real lending decision engine works, including the explainability a credit decision legally and practically requires.

**Dataset:** Statlog German Credit Data (UCI Machine Learning Repository, Hofmann 1994) — 1,000 loan applicants, 20 attributes.

## Problem

When a bank or lender receives a loan application, it has to decide: approve or decline? Get it wrong in either direction and it costs money — approving a bad-risk applicant risks the loan defaulting, while declining a good-risk applicant means lost business. Unlike many prediction problems, a credit decision also has to be **explainable**: a lender needs to be able to justify *why* an applicant was declined, not just output a probability.

## What this project does

- **Cleans and decodes** the raw Statlog dataset (originally coded as `A11`, `A34`, etc.) into human-readable categories
- **Engineers 7 finance-relevant features** — debt-to-duration ratio, installment burden, age brackets, loan term flags — that reflect actual credit-risk reasoning, not just raw columns
- **Trains and compares 3 models** (Logistic Regression, Random Forest, XGBoost), selecting the best by ROC-AUC
- **Adds SHAP explainability** — for every single prediction, the dashboard shows exactly which factors pushed the decision toward "bad risk" or "good risk," and by how much
- **Serves live predictions** through an interactive Streamlit dashboard with a portfolio overview, a live applicant scoring tool, and a model comparison view

## Results

| Model | Accuracy | Precision (Bad Risk) | Recall (Bad Risk) | ROC-AUC |
|---|---|---|---|---|
| Logistic Regression | 72% | 0.52 | 0.82 | 0.799 |
| **Random Forest** | 72% | 0.53 | 0.72 | **0.806** |
| XGBoost | 69% | 0.49 | 0.63 | 0.762 |

Random Forest was selected as the production model based on ROC-AUC. Recall on "Bad Risk" applicants is weighted heavily in evaluation, reflecting the original Statlog cost matrix: misclassifying a bad-risk applicant as good is penalized 5x more heavily than the reverse, because that's the costlier mistake for a lender.

**Key insight surfaced by SHAP:** checking account status, loan duration, and installment burden (installment rate × credit amount) are the strongest drivers of predicted default risk — applicants with no visible checking account history and long, high-installment loans carry the highest risk.

## Tech stack

Python · Pandas · NumPy · Scikit-learn · XGBoost · SHAP · Streamlit · Matplotlib

## Project structure

```
credit_risk_project/
├── data/credit_risk.csv        # Raw Statlog German Credit data
├── src/train_model.py          # Cleaning, decoding, feature engineering, training, SHAP
├── src/app.py                  # Streamlit dashboard (overview + live scoring + SHAP explanation)
├── models/                     # Saved model, scaler, encoders, SHAP explainer
├── outputs/metrics.json        # Evaluation results (generated by train_model.py)
└── requirements.txt
```

## Run it locally

```bash
pip install -r requirements.txt
python src/train_model.py     # trains models, computes SHAP, saves artifacts
streamlit run src/app.py      # launches the dashboard at localhost:8501
```
